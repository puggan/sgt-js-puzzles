Puzzles are fun

TLDR: Want to build your own puzzles, use the JSON-api to get generated puzzles, just add an UI.

I love playing puzzle-games, I can do them on paper, but I prefer an friendly UI that helps me keep track of clues.

Simon G. Tatham (SGT) Puzzle collection have allot of good puzzles, and often a good UI.

I also plays nonograms(/pattern) with non-random images, and Zebra-puzzles

In this project I used Simons code to generate puzzle-states, and then converting them to JSON-format, and then addin my own UI.

Api

In the API-folder you find a few php-script, that genereates a game-state in JSON-format.

Thus can be used to implement your own UI, or to build a solver.

So far I have implemented an JSON-format for 15 of the puzzles.

Code / Repo

Here is the source-code for all parts of the project:

  • SGT Puzzle collection [c]: The original code from Simon.
  • Patched SGT Puzzles [c]<: My modification, to make the standalone solvers output a single game-state that i can parse.
  • JSON-API [php]: My code to parse the solvers output and reformat it as JSON
  • My Puzzles [js]: My implementation of a UI to the puzzles
